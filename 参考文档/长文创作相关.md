很好，这里是一些关于“利用 agent （多智能体）构建长文/长篇创作”方向的优秀论文，建议你可以选几篇作为起点，并基于你要做 agent 的具体任务（例如：写小说？撰写研究报告？生成技术文档？）再进一步搜集相关文献。以下列出 **5 篇代表性论文**，附摘要 +说明为什么值得阅读。

---

1. **LongAgent: Scaling Language Models to 128k Context through Multi‑Agent Collaboration** (Zhao et al., 2024)

   * 摘要：该论文提出 “LongAgent” 框架，通过多智能体协作方式，将 LLM 用于超大上下文（128k token）任务。一个 “leader agent” 引导多个“成员 agent”去处理信息，然后成员之间进行通信解冲突。 ([arXiv][1])
   * 为什么值得：如果你做的 agent 是要处理 **非常长的文档**或**长篇创作**（比如书、报告、章篇结构），“如何处理长上下文”是关键难点。本文给出了技术路线。
   * 关联建议：你可以考虑类似“leader-agent + worker-agents +通信机制”的架构。

2. **Beyond Outlining: Heterogeneous Recursive Planning for Adaptive Long‑form Writing with Language Models** (Xiong et al., 2025)

   * 摘要：针对长文写作代理（writing agents）提出一种 **异构递归规划（heterogeneous recursive planning）** 框架，将检索／推理／写作任务类型动态整合，而非预设固定流程。 ([arXiv][2])
   * 为什么值得：如果你的 agent 不仅“写”，还要“检索资料”“推理”“构思结构”，这个框架比较贴近“从0到1”搭建写作代理的场景。
   * 关联建议：可以参考其任务分解机制，设计你的 agent 模块（如 Research → Plan → Write）。

3. **SuperWriter: Reflection‑Driven Long‑Form Generation with Large Language Models** (Wu et al., 2025)

   * 摘要：该论文提出 “SuperWriter-Agent” 框架，模拟人类作家“思考→大纲→写作→精炼”流程，通过规划、写作、精炼三个阶段。还用了 DPO（Direct Preference Optimization）＋ MCTS（蒙特卡洛树搜索）优化。 ([ResearchGate][3])
   * 为什么值得：如果你要做的是**长篇创作**（例如小说、技术报告、深度文章）且希望质量高，这篇论文所强调的「精炼／反思／结构」机制很有借鉴价值。
   * 关联建议：在你的 agent 中加入“反思／审阅”模块可能会显著提升输出质量。

4. **A Cognitive Writing Perspective for Constrained Long‑Form Text Generation** (Wan et al., 2025)

   * 摘要：提出 “CogWriter” 框架，从认知写作角度出发：一个 Planning Agent 执行分层规划，多个 Generation Agents 并行执行计划，同时有质量监控／审阅机制。 ([ACL Anthology][4])
   * 为什么值得：如果你希望你的 agent 系统不仅能“生成”而且能“管理流程”“监控质量”，这篇提供很好的思路。
   * 关联建议：可考虑设计一个总 agent 作为“督导”，其他 agent 作为“执行”，形成流程闭环。

5. **NexusSum: Hierarchical LLM Agents for Long‑Form Narrative Summarization** (Kim & Kim, 2025)

   * 摘要：该研究面向“长篇叙事（书籍／电影／剧本）摘要”任务，提出 NexusSum —— 三阶段 pipeline ：对话-转描述预处理、层级多 LLM 总结、迭代压缩。 ([arXiv][5])
   * 为什么值得：虽然重点是“摘要”而不是“创作”，但其处理长文本、分阶段、层级结构的思路也可移植。若你的 agent 要“生成”而非“摘要”，仍可参考其大框架。
   * 关联建议：可以参考其“预处理（dialogue→描述）”的技巧，以提升上下文一致性。

### ⚠️ 改造建议 &你需要注意的点

* 要加入 **剧本专有结构**：角色（名字／性格／目标）、场景（地点／时间／背景）、对白／动作、镜头指示（可选）

* 要处理 **格式化输出**：生成剧本格式（Scene 1、Act 2、INT./EXT. 等）更专业
* 要规划 **流程**：如 提示／大纲 → 场景拆分 → 对白＋动作 → 校对／润色 → 最终整合。
* 如果你使用 TypeScript／JavaScript （你前端背景）而不是 Python，也可以把这些项目作为概念参考，再自己用 JS 生态改写。
* 要考虑 **API／模型成本**：多 agent 系统调用频繁、反复修正，成本可能高。要控制 prompt 长度、模型调用次数。
* 要考虑 **版权／敏感内容**：生成剧本可能涉及角色名字／背景版权问题。校对 agent 要检测重复、侵权风险。
