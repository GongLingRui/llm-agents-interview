### 如何写出真正好用的 Skill
**说明**：让问题驱动建设，分层设计，与 Claude 共创。

#### ① 从评估开始：问题驱动
- 先让 Claude 执行真实任务，记录出错点/卡顿点。
- 每个 Skill 都针对具体缺口，小步验证、逐步扩展。

#### ② 为规模而设计：分层结构
- 主流程写在 SKILL.md，细节拆到 forms/reference 等文件。
- 元数据 (~100 tokens) → 正文 (<5k) → 附录 (unlimited)。

#### ③ 从 Claude 的视角命名与描述
- `name + description` 决定 Claude 是否触发技能，越精准越好。
- 监控实际使用：是否误触或过度依赖，作为迭代依据。

#### ④ 与 Claude 一起迭代
- 成功时让 Claude 总结“哪些做法有效”，写回 Skill。
- 失败时引导它反思缺口，再补入 SOP，形成人机共创的学习闭环。

### 安全：让 Skill 可信可控
**说明**：Skill 能运行脚本、访问文件系统，安全意识必须同步升级。

#### 三条安全红线
- **只安装可信来源**：不要执行来历不明的技能包或脚本。
- **使用前逐项检查**：仔细阅读技能内所有文件，尤其是脚本、依赖、外部链接。
- **理解执行逻辑**：分清哪些代码要跑、运行前的输入输出是什么，避免误触。

#### 安全 ≠ 限制能力，而是保护信任
Skill 的开放性让 Claude 真能“做事”，但也意味着更大的攻击面。只有把安全当工程规范，而不是事后补丁，才能让团队放心让 Claude 处理真实资产。

### 未来：技能网络与人机共生
**说明**：从个人化 SOP 到共享技能库，再到 Agent 自我成长。

#### 共享与发现
Skill 已在 Claude.ai、Claude Code、Agent SDK 等渠道上线。未来，每个人/组织都能创建、编辑、共享技能，形成完整生命周期的“技能生态”。

#### Skill × MCP：思维 + 执行动作
Skill 提供任务知识，MCP（Model Context Protocol）提供与数据库/API/软件的连接。当两者结合，Claude 既懂得如何完成任务，又能真正动手执行。

#### Agent 自我进化
Anthropic 的愿景：未来的 Agent 能自主创建、编辑并评估自己的 Skill，把反复验证的思维与操作沉淀为可复用的能力模块——Skill 成为智能体写给自己的成长记录。
