### 一、拥抱失败：构建健壮Agent的反直觉智慧
- **问题定义：重新定义Agent的“错误”**
  - **常规做法的冲动与代价**
    - 开发者本能：隐藏错误（清理痕迹、默默重试）。
    - 致命代价：擦除失败，就等于移除了模型从中学习的证据。没有证据，模型就无法适应和改正。
  - **核心前提：错误不是Bug**
    - Agent犯错是必然的，是其行动循环的一部分。来源包括：模型幻觉、环境错误、工具异常等。
    - 心态转变：我们的目标不应是“永不犯错”，而是“擅长从错误中恢复”。


### 二、Manus的做法：将失败作为学习的证据
- **核心原则：将错误的尝试完整地保留在上下文中**
  - **错误做法：隐藏失败**
    流程示例：`Instruction` → `Action 1` → `Observation 1` → `Action 2 (Success)` → `Observation 2 (Success)`
    后果：模型永远不知道自己错过。
  - **正确做法：保留失败**
    流程示例：`Instruction` → `Action 1` → `Observation 1` → `Action 2A (Fail)` → `Observation 2A (Error)` → `Action 2B (Corrected)` → `Observation 2B (Success)`
    效果：模型看到错误并从中学习。
- **工作原理**：当模型看到失败记录时，会隐式地更新其内部信念，从而极大地降低重复犯错的可能性。


### 三、最终收益与启示
- **对产品和开发的启示**
  - 产品经理：将“错误恢复”作为Agent的核心功能和关键体验指标来设计。
  - 开发者：避免在循环中自动清理失败记录，确保失败的证据链得以保留。
- **衡量真正的“智能”**
  - Manus认为，一个Agent能否**从错误中恢复 (Error Recovery)**，是衡量其是否具备**真正智能行为 (Agentic Behavior)**的最明显指标之一。
- **最终结论**：构建健壮的Agent，关键不在于掩盖失败，而在于教会它如何“看到”、“记录”，并最终从失败中学习。
